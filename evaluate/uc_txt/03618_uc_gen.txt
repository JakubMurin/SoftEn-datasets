1. User initiates match request.
2. API Gateway receives the request.
3. Controller processes the request.
4. Service checks for a match.
5. MatchRequests are retrieved for the user.
6. Another MatchRequest is retrieved.
7. A random question is fetched.
8. SessionInfo is created and attached to MatchRequests.
9. MatchRequests are updated.
10. User's MatchRequest is sent back.
11. User switches to a session page.
