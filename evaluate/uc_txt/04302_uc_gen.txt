1. User navigates to 'Submit Session'.
2. Web Interface displays 'Session Submission Form'.
3. User fills in and submits the form.
4. Session details are saved in the database.
5. Session Review Process is triggered.
6. Session status changes to 'Under Review'.
7. Notification is triggered to the user.
8. User receives an email notification about 'Session Submitted'.
9. User refreshes/checks email.
10. Session status is updated to 'Approved' or 'Rejected'.
11. Notification is triggered to the user about evaluation.
12. User receives an email notification about 'Session Approved' or 'Session Rejected'.
