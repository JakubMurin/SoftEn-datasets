1. Users initiate a transaction with Merchant.
2. Merchant initiates a transaction with Vipps.
3. Vipps provides a session link to Merchant.
4. Merchant triggers a transaction session in UserVippsApp.
5. UserVippsApp approves payment and details sharing with Vipps.
6. Vipps saves Consent and generates a sub.
7. Vipps performs the payment.
8. Vipps provides a response to UserVippsApp.
9. UserVippsApp provides a response to Merchant.
10. Merchant retrieves details from Vipps.
11. Vipps returns details, including sub, to Merchant.
12. Merchant gets user information from Vipps.
13. Vipps returns user information to Merchant.
14. Merchant handles the session result.
