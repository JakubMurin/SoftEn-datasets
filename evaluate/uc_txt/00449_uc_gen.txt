1. HydratorUser selects the option to create a new pipeline.
2. Web UI displays the pipeline creation form.
3. HydratorUser fills in pipeline details and specifies a new dataset instance.
4. Web UI initiates pipeline and dataset creation.
5. Pipeline requests dataset type check.
6. Dataset Management Service verifies dataset type and version.
7. Version Control provides dataset type specifications.
8. Dataset Management Service confirms dataset type.
9. Pipeline requests creation of new dataset instance.
10. Dataset Instance Creator creates a new dataset instance.
11. Dataset Management Service registers new dataset instance version.
12. Dataset Instance Creator confirms dataset instance creation.
13. Pipeline displays success message and pipeline details on Web UI.
