1. Пользователь кликает на кнопку "Создать коллбек" в интерфейсе.
2. Web UI отправляет запрос на создание коллбека с указанием имени и описания.
3. Callback GW получает запрос и обращается к Token API для валидации API ключа.
4. Token API возвращает подтверждение верности ключа для GW.
5. GW запрашивает состояние клиента у CMA.
6. CMA возвращает статус клиента как "активный" GW.
7. GW создает коллбек в Resource Inventory по указанному уникальному идентификатору приложения и коллбека.
8. Resource Inventory определяет и возвращает идентификатор созданного коллбека обратно GW.
9. GW запрашивает атрибуты созданного коллбека в Resource Inventory.
10. Resource Inventory возвращает атрибуты приложения для GW.
11. GW добавляет новую запись в Action History указывая успех операции с кодом 200 OK.
12. GW отправляет идентификатор созданного коллбека обратно в Web UI.
13. Web UI выводит информацию о новом созданном коллбеке пользователю.
