1. Collection Curator selects item(s) for expiration through the Web Interface.
2. Web Interface sends the selected items and expiration parameters to the EDRMS Server.
3. EDRMS Server queries and updates the item(s) with the specified expiration date in the Repository Database.
4. EDRMS Server schedules tasks for the expiration of the selected item(s).
5. UI confirms back the update indicating the successful setting of the expiration date.
6. Web Interface displays a confirmation message to the Collection Curator.
