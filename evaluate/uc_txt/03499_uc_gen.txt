1. The faculty member selects a data file for upload.
2. The web browser interface initiates the file upload process.
3. The file upload service stores the data file in the digital repository.
4. The database saves the file metadata, including uploader information.
5. The database confirms the metadata storage.
6. The digital repository confirms the file storage.
7. The file upload service displays the file upload success to the faculty member.
8. The file upload service provides the file ID and upload confirmation to the faculty member.
9. The faculty member chooses a licensing option (CC-By or CC0) in the web browser interface.
10. The web browser interface sends the license choice and file ID to the licensing service.
11. The licensing service applies the chosen license to the file.
12. The DRM service updates the file metadata with the license information.
13. The database updates the file record with the license metadata.
14. The database confirms the license update.
15. The digital repository confirms the license application to the file.
16. The licensing service displays the license application success to the faculty member.
17. The faculty member confirms that the license has been set.
