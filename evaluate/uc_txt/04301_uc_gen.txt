1. User navigates to 'Submit Session' on the Web Interface.
2. Web Interface displays 'Session Submission Form' to the Authenticated User.
3. User fills in and submits the form on the Web Interface.
4. Session details are saved in the Session Database.
5. Session Review Process is triggered by the Session Database.
6. Session status is updated to 'Under Review' in the Session Database.
7. Notification to the User is triggered by the Session Database.
8. Email notification 'Session Submitted' is sent to the User by the Notification Service.
9. User refreshes/checks email for notification.
10. Session status is updated to 'Approved/Rejected' in the Session Database by the Session Review System.
11. Notification to the User is triggered by the Session Database.
12. Email notification 'Session Approved' or 'Session Rejected' is sent to the User by the Notification Service.
