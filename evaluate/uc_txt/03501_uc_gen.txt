1. Cornell Faculty Member selects data file for upload.
2. Web Browser Interface initiates file upload process.
3. File Upload Service stores data file in Digital Repository.
4. Database saves file metadata, including uploader info.
5. Database confirms metadata storage.
6. Digital Repository confirms file storage.
7. File Upload Service displays file upload success to Web Browser Interface.
8. File Upload Service provides file ID and upload confirmation to Cornell Faculty Member.
9. Cornell Faculty Member chooses licensing option (CC-By or CC0) in Web Browser Interface.
10. Web Browser Interface sends license choice and file ID to Licensing Service.
11. Licensing Service applies chosen license to file using Digital Rights Management Service.
12. Digital Rights Management Service updates file metadata with license info in Digital Repository.
13. Database updates file record with license metadata.
14. Database confirms license update.
15. Digital Repository confirms license application to file with Licensing Service.
16. Licensing Service displays license application success to Web Browser Interface.
17. Web Browser Interface confirms license has been set to Cornell Faculty Member.
