1. USER requests VC offer from TAS.
2. TAS sends VC offer response to USER.
3. CA confirms VC offer to USER.
4. CA requests VC Plan ID list from TAS.
5. TAS returns VC Plan ID list response to CA.
6. CA issues VC start request to TAS.
7. TAS sends issue VC start response.
8. CA sends ECDH request to TAS.
9. TAS responds with ECDH response to CA.
10. CA requests issue Profile from TAS.
11. TAS returns issue Profile response to CA.
12. CA requests issue VC from WALLET.
13. WALLET sends issue VC request to TAS.
14. TAS processes VC issuance.
15. ISSUER registers VC meta in Blockchain.
16. ISSUER completes VC E2E encryption.
17. TAS sends VC response to WALLET.
18. WALLET decrypts VC response.
19. WALLET stores the VC.
20. CA requests VC completion from TAS.
21. TAS completes VC issuance and sends response to CA.
