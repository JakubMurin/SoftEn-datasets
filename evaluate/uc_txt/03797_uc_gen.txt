1. User requests User Registration Start through the Certified App.
2. ECDH Request for key agreement process initiated between CA, Wallet, and TAS.
3. Server Token Creation requested by CA through TAS.
4. User performs KYC to register personal identifiers. [Not specified in this specification].
5. Wallet generates keys, DID Document, creates User Registration Request and submits to TAS.
6. TAS verifies, registers DID, stores mapping information, and responds User Registration Request. TAS Interaction with Blockchain.
7. User Registration Completion Request sent through the Certified App to TAS.
8. TAS validates and finalizes the registration, sending User Registration Completion Response.
