1. Depositor accesses the platform.
2. UI sends a request for authentication.
3. Auth authenticates the Depositor.
4. Depositor provides credentials to Auth.
5. Auth confirms successful authentication to UI.
6. Depositor requests a familiar interface.
7. UI fetches user preferences.
8. SI requests user-specific settings from UnivSys.
9. UnivSys retrieves and sends user settings to SI.
10. SI applies settings and customization to UI.
11. UI requests integration features.
12. SI fetches integration capabilities from UnivSys.
13. UnivSys provides capabilities (APIs, data formats) to SI.
14. Integration with University systems carried out by SI.
15. DMP renders a unified management interface.
16. UI displays a familiar and integrated interface for the Depositor.
