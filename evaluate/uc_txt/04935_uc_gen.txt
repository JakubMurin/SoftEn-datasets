1. Admin requests to view settings list.
2. List Settings View sends request to Settings Controller.
3. Settings Controller checks data.
4. Settings Controller retrieves settings list from Settings Model.
5. Settings Model returns settings list to Settings Controller.
6. Settings Controller sends settings list result to List Settings View.
7. List Settings View displays settings list to Admin.
8. Admin edits settings and saves changes.
9. List Settings View sends new settings data to Settings Controller.
10. Settings Controller checks data.
11. Settings Controller saves settings in Settings Model.
12. Settings Model returns save result to Settings Controller.
13. Settings Controller sends updated settings list result to List Settings View.
14. List Settings View displays updated settings list to Admin.
