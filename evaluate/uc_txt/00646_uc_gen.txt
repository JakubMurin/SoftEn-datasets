1. User sends a Reconfigure Dataset Request to the Dataset Management System.
2. The Dataset Management System validates the Explore Table Update with the ExploreDB.
3. If the Explore Table Update is successful:.
4. The Dataset Management System proceeds with the Dataset Reconfiguration.
5. The Dataset Management System informs the user about the Reconfiguration Success.
6. If the Explore Table Update fails:.
7. The Dataset Management System informs the user about the Reconfiguration Failure.
8. The Dataset Management System rolls back any changes made during the process.
