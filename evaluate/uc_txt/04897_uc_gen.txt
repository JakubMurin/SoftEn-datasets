1. Admin requests to view settings.
2. List Settings View sends request to Settings Controller.
3. Settings Controller checks data and retrieves settings list from Settings Model.
4. Settings Controller sends settings list result to List Settings View.
5. List Settings View displays settings list to Admin.
6. Admin edits settings and saves.
7. List Settings View sends new settings data to Settings Controller.
8. Settings Controller checks data and saves settings in Settings Model.
9. Settings Controller sends save result to List Settings View.
10. List Settings View displays updated settings list to Admin.
