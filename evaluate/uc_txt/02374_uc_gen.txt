1. Administrator asks UI to create a new type of test.
2. UI lists parameter categories by calling getParameterCategories().
3. UI shows parameter category list and asks Administrator to select one.
4. Administrator selects a parameter category.
5. UI requests data (description, collecting method, code).
6. Administrator types the requested data.
7. UI creates a test type with the entered data.
8. UI shows all data and requests confirmation.
9. Administrator confirms all data.
10. UI informs Administrator about the operation success.
