1. Admin requests to view the list of settings.
2. List settings view sends request.
3. Settings controller checks data.
4. Settings controller gets the list of settings.
5. Settings model returns the list of settings.
6. Settings controller returns the list of settings to list settings view.
7. List settings view displays the list of settings.

11. Settings controller saves settings.

13. Settings controller returns list of settings result to list settings view.
