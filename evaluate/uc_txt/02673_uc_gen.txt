1. Developer deploys a new dataset type version.
2. Dataset Management System checks the current version from Version Control.
3. Version Control returns the current version details to Dataset Management System.
4. Dataset Management System registers the new version in the Dataset Type Registry.
5. Dataset Type Registry confirms successful registration to Dataset Management System.
6. Dataset Management System updates the available versions list with Dataset Instance Manager.
7. Dataset Instance Manager confirms the list update to Dataset Management System.
8. Dataset Management System isolates the impact of the new version in the Dataset Type Registry.
9. Dataset Type Registry confirms the impact isolation to Dataset Management System.
10. Dataset Management System creates a branch for the new version in Version Control.
11. Version Control confirms the successful branch creation to Dataset Management System.
12. Dataset Management System assigns the new version to new instances through Dataset Instance Manager.
13. Dataset Instance Manager confirms the usage of the new version in new instances to Dataset Management System.
14. Dataset Management System informs the Developer about the successful deployment of the new version.
