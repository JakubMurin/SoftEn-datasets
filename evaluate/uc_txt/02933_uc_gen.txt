1. Depositor accesses the platform.
2. UI requests authentication from Auth.
3. Auth authenticates the Depositor.
4. Depositor provides credentials to Auth.
5. Auth confirms successful authentication to UI.
6. Depositor requests a familiar interface from UI.
7. UI fetches user preferences from SI.
8. SI requests user-specific settings from UnivSys.
9. UnivSys provides the user settings to SI.
10. SI applies settings and customization to UI.
11. UI requests integration features from SI.
12. SI fetches integration capabilities from UnivSys.
13. UnivSys provides capabilities (APIs, data formats) to SI.
14. SI integrates with University Systems.
15. UI renders a unified management interface through DMP.
16. DMP indicates the interface is ready to UI.
17. UI displays a familiar and integrated interface to the Depositor.
