1. Depositor accesses the platform.
2. UI requests authentication from Auth service.
3. Auth service authenticates Depositor.
4. Depositor provides credentials to Auth service.
5. Auth service confirms authentication to UI.
6. UI requests familiar interface customization from SI.
7. SI fetches user preferences from UnivSys.
8. UnivSys provides user settings to SI.
9. SI applies settings and customization to UI.
10. UI requests integration features from SI.
11. SI fetches integration capabilities from UnivSys.
12. UnivSys provides integration capabilities to SI.
13. SI integrates with University Systems.
14. UI renders unified management interface via DMP.
15. DMP signals readiness of the interface to UI.
16. UI displays familiar and integrated interface to Depositor.
